# app-hub 需求说明

## 项目定位

app-hub 是一个统一的应用集合平台，为用户提供游戏、工具、艺术、音频、视觉特效等多分类应用的统一入口。

## 核心需求

### 1. 统一入口
- 所有子项目通过单一域名访问
- 外网 URL：`https://97.383636.xyz/code/20008/<项目名>/`
- 内网 URL：`http://localhost:20008/<项目名>/`
- 大厅首页展示所有可用项目

### 2. 项目发现
- 后端自动扫描 `apps/` 目录发现新项目
- 支持手动触发重新扫描（`POST /api/_rescan`）
- 项目元数据管理（名称、描述、图标、分类）

### 3. 路由分发
- 统一 API 前缀：`/api/<项目名>/`
- 支持 Express Router 和 WebSocket
- 项目路由热加载

### 4. 静态资源服务
- 子项目构建产物放在 `static/<项目名>/`
- 自动导航栏注入
- 路径遍历防护

### 5. 数据统计
- 项目点击统计（SQLite 存储）
- 健康检查接口（含项目列表和统计）

## 非功能需求

### 安全
- CSRF 防护（Cookie + Token 双验证）
- 速率限制（API 100次/分钟/IP）
- 路径遍历防护（realpath 校验）
- 输入验证

### 性能
- SQLite WAL 模式
- 静态文件缓存（ETag/Last-Modified）
- 点击统计缓存（30秒 TTL）

### 可靠性
- 优雅退出（SIGTERM/SIGINT）
- 全局异常捕获
- 数据库连接管理

## 技术约束

- Node.js + TypeScript
- Express 5（ESM）
- better-sqlite3（WAL 模式）
- Vite 8 + TypeScript 5.9（前端）
- Vitest（测试）
