## 一、架构总览

应用大厅按三层分离职责：

| 层级 | 目录 | 定位 |
|------|------|------|
| **前端层** | `lobby-web/` | 大厅前端壳：静态页面服务、导航栏注入、WebSocket 代理 |
| **业务层** | `apps/<分类>/<项目名>/` | 各子项目独立业务，104 个前端应用 |
| **后端层** | `hub-server/` | 轻量共享后端：统一 API、项目发现、路由分发、数据统计 |
| **共享包** | `packages/` | app-utils（前端工具库）+ design-system（设计系统） |
| **部署产物** | `static/` | 构建输出目录，`static/index.html` 为大厅入口 |

## 四、hub-server 架构

### 源文件结构

```
hub-server/src/
├── index.ts                     # 入口：Express + HTTP Server 启动
├── config.ts                    # 配置（端口、路径、环境变量）
├── router.ts                    # 路由发现、加载、重扫描
├── route-loader.ts              # 项目路由模块动态加载器
├── http-stack.ts                # HTTP 中间件栈（CSRF/限流/静态/API）
├── static-site.ts               # 静态站点服务 + 导航栏注入
├── static-ui.ts                 # 导航栏 HTML 模板（右上角浮动）
├── lobby-snapshot.ts            # 大厅初始化/引导快照构建
├── project-catalog.ts           # 项目目录发现与元数据管理
├── project-meta.json            # 项目元数据（图标、名称、描述）
├── db.ts                        # SQLite 数据库连接管理（WAL 模式）
├── logger.ts                    # 日志工具
├── route-health.ts              # 健康检查路由注册
├── upgrade-listener-registry.ts # WebSocket 升级监听器注册表
├── integrations/
│   └── app-lobby.ts            # app-lobby 集成层（CORS 中间件）
├── types/                       # 类型定义
│   ├── constants.ts             # 游戏常量枚举
│   ├── game.ts                  # Player/Room/GameMessage 接口
│   └── messages.ts              # 游戏消息数据接口
├── routes/                      # 项目路由（34 个）
│   ├── basketball/
│   ├── battleship/
│   ├── ...（共 34 个）
│   └── xiangqi/
└── __tests__/                   # 测试文件（7 个）
    ├── db.test.ts
    ├── http-stack.test.ts
    ├── lobby-snapshot.test.ts
    ├── project-catalog.test.ts
    ├── route-loader.test.ts
    ├── static-site.test.ts
    └── upgrade-listener-registry.test.ts
```

### 核心配置（config.ts）

```typescript
export const PORT = parseInt(process.env.PORT || '20008', 10);
export const STATIC_ROOT = process.env.STATIC_ROOT || '/root/projects/app-hub/static';
export const HOME_URL = process.env.HOME_URL || '/openclaw/20008/';
export const STATIC_DISCOVERY_TTL_MS = parseInt(process.env.STATIC_DISCOVERY_TTL_MS || '60000', 10);
```

### 路由加载机制

1. **启动时**：`router.ts` 扫描 `src/routes/` 目录，排除 `admin`, `auth`, `platform`, `platform-console`
2. **加载时**：`route-loader.ts` 动态 import 项目模块，注册 Express Router 和 WebSocket
3. **热重载**：`POST /api/_rescan` 触发重新扫描（需 Admin API Key）

### 静态文件服务

| 路由 | 处理 |
|------|------|
| `GET /` | 返回 `static/index.html` |
| `/app-lobby/*` | `express.static` 大厅 JS/CSS 资源 |
| `/<项目名>/*` | `express.static` 子项目静态文件 |
| HTML 页面 | 自动注入导航栏 + CSRF Token |


路径规则：`/sio/<project>/websocket` → `ws://<backend>/api/<project>/websocket`

## 八、反向代理架构

```
用户浏览器
    │
    ▼
Nginx (443/80)
    │
    ▼
hub-server (:20008)
    ├── /           → static/index.html
    ├── /app-lobby/ → express.static
    ├── /api/       → API 路由
    ├── /<项目名>/  → express.static
    └── WebSocket   → 项目级 WebSocket
```

Nginx 配置关键项：
- `proxy_pass` 到 `http://127.0.0.1:20008`
- `proxy_set_header X-Forwarded-For` 传递真实 IP
- `proxy_set_header Upgrade` / `Connection` 支持 WebSocket

## 九、WebSocket 实现

### 升级流程

1. 客户端连接到 `/api/<project>/websocket`
2. `upgrade-listener-registry.ts` 管理 upgrade 事件监听器
3. 各项目的 `socketSetup(httpServer, wsPath)` 注册监听
4. 使用 `Proxy` 包装 HttpServer，拦截 `on('upgrade')` 事件

### 项目路由

- 联机项目以 `-mp` 后缀标识（如 `game-2048-mp`, `tetris-mp`）
- WebSocket 路径：`/api/<project>/websocket`
- 大厅不展示 `-mp` 入口，仅在主项目内作为能力存在

## 十、安全机制

### CSRF 防护

- 安全方法（GET/HEAD/OPTIONS）：生成/刷新 CSRF Token Cookie
- 非安全方法：验证 Cookie Token 与 Body/Header Token 一致性
- 使用自定义实现（替代已废弃的 csurf）

### 速率限制

- API 接口：100 次/分钟/IP
- 使用 `express-rate-limit`

### 路径遍历防护

- 使用 `realpathSync` 解析符号链接
- 校验解析后的路径是否在 `STATIC_ROOT` 范围内

### 全局错误处理

- `uncaughtException`：记录日志后退出
- `unhandledRejection`：记录日志后退出
- Express 错误中间件：统一返回 500

### 优雅退出

- `SIGTERM` / `SIGINT`：关闭数据库连接 → 关闭 HTTP 服务器 → 退出
- 10 秒强制退出超时
