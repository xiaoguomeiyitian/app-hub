# 麻将 — 需求文档

> 📅 创建日期: 2026-04-01
> 📁 项目目录: `/root/projects/apps/mahjong/`

## 项目概述

在线麻将游戏，支持国标规则和川麻血战到底两种玩法，单机 AI 对战 + 在线多人对战。

## 功能需求

### P0 — 核心功能

| # | 功能 | 描述 | 验收标准 |
|---|------|------|---------|
| F01 | 牌组 | 144 张（万/条/筒各 36 + 字牌 28 + 花牌 8）| 川麻模式去掉花牌=108 张 |
| F02 | 洗牌砌牌 | 随机洗牌，4 方各砌牌墙 | 牌墙正确 |
| F03 | 摸牌打牌 | 轮流摸牌→打牌基本循环 | 流程正确 |
| F04 | 吃碰杠 | 上家可吃，任意家可碰/明杠/暗杠/补杠 | 优先级：胡 > 杠 > 碰 > 吃 |
| F05 | 胡牌判定 | 基本胡型：顺子+刻子+将（国标） | 正确判定胡牌 |
| F06 | 特殊胡型 | 七对、十三幺、全不靠等 | 国标常见胡型全覆盖 |
| F07 | 番型计算 | 国标 81 番（大四喜/大三元/绿一色等） | 番数计算正确 |
| F08 | AI 对战 | 3 家 AI，有基本出牌策略 | AI 可完整对局 |
| F09 | 川麻血战 | 血战到底规则：胡了继续打，直到流局或全部胡完 | 规则正确 |
| F10 | 牌面绘制 | 万/条/筒/字/花 精美牌面 | 可辨识，美观 |

### P1 — 增强功能

| # | 功能 | 描述 | 验收标准 |
|---|------|------|---------|
| F11 | 听牌提示 | 标记可胡的牌 | 用户可查看听牌列表 |
| F12 | 摆牌动画 | 摸牌/打牌/吃碰杠动画 | 流畅 |
| F13 | 战绩统计 | 历史对局记录 | SQLite 存储 |
| F14 | 规则切换 | 大厅选择国标/川麻 | 切换后规则生效 |

### P2 — 联机功能

| # | 功能 | 描述 | 验收标准 |
|---|------|------|---------|
| F15 | 好友房 | 创建房间码，4 人加入 | 4 人通过房间码进入同局 |
| F16 | 匹配 | 4 人自动匹配 | 15 秒内匹配成功 |
| F17 | 实时同步 | WebSocket 同步摸牌/打牌/吃碰杠 | 延迟 < 200ms |
| F18 | 断线重连 | 掉线 60 秒内可重连 | 恢复手牌和游戏状态 |
| F19 | 机器人补位 | 缺人时 AI 接管 | 无感知切换 |

## UI 要求

| 项目 | 要求 |
|------|------|
| 风格 | 东方美学，木质桌面，牌面精美 |
| 牌面 | 万/条/筒用传统汉字+花色，字牌用红/绿/黑 |
| 座位 | 四方位，己方在底部，对家在顶部 |
| 手牌 | 横排展示，选中牌上移 |
| 出牌区 | 中央区域，打出的牌按时间排列 |
| 吃碰杠 | 显示在侧面，标记来源 |

## 设备适配（通用标准）

| 分辨率 | 布局策略 |
|--------|---------|
| 375-412px（手机） | 己方手牌底部横向滑动，对家/左家/右家牌缩小在对应方位，出牌区居中 |
| 768-1024px（平板） | 四方位展开更大，出牌区增加空间 |
| 1280-1920px（桌面） | 经典四方桌布局，牌面清晰 |

**适配验收清单**：
- [ ] 375/390/412/768/1024/1280/1366/1920 全部无溢出/重叠/截断
- [ ] 触控按钮 ≥44pt
- [ ] 手牌在最小屏可滑动浏览
- [ ] 吃碰杠按钮在小屏不遮挡手牌
- [ ] 文字 ≥12px 可读（牌面汉字清晰）

## 技术约束

| 项目 | 要求 |
|------|------|
| 前端 | Vite + TypeScript strict + `base: './'` |
| 后端路由 | `shared-backend/src/routes/mahjong/` |
| WebSocket | 原生 `ws` `noServer` 模式 |
| 数据库 | `better-sqlite3`（战绩+番型记录） |
| 部署 | `cp -r dist/* /root/projects/static/mahjong/` |
