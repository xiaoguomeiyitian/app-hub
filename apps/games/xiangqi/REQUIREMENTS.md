# 需求文档 — 中国象棋

> 📅 版本日期: 2026-03-31
> 📁 项目目录: `/root/projects/apps/xiangqi`

## 1. 项目概述

中国象棋游戏，支持单机 AI 对战、残局挑战、在线匹配对战、好友房间对战、历史复盘与观战。

## 2. 功能需求

### P0 — 核心功能（必须实现）

| # | 功能 | 描述 | 验收标准 |
|---|------|------|----------|
| F01 | 棋盘渲染 | Canvas 绘制标准中国象棋棋盘（9×10） | 棋盘、网格、楚河汉界、九宫格正确显示 |
| F02 | 棋子绘制 | 32 枚棋子（红黑各 16 枚），含汉字标识 | 棋子颜色、文字、位置正确 |
| F03 | 走棋规则 | 7 种棋子走法合法性验证 | 所有走法规则正确执行 |
| F04 | 吃子 | 走到对方位置吃掉对方棋子 | 吃子后棋盘状态正确更新 |
| F05 | 单机对战 AI | 纯前端本地 AI 对战，支持红/黑执棋选择 | 选择颜色后可与 AI 完整对局 |
| F06 | 将军检测 | 走棋后判断是否"将军" | 将军时有明确提示 |
| F07 | 胜负判定 | 将/帅被吃或无路可走 | 游戏正确结束并显示结果 |
| F08 | 视角固定 | 无论玩家执红还是执黑，玩家势力均显示在下方 | 玩家一侧始终位于底部，敌方位于顶部 |
| F09 | 模式选择 | 大厅页支持选择单机、残局、在线、好友、历史入口 | 各模式入口均可正常进入 |
| F10 | 在线匹配 | WebSocket 匹配队列 + 时间档位匹配 | 玩家可匹配到对手开始对局 |
| F11 | 在线对战 | 实时 WebSocket 走棋同步 | 双方走棋实时可见 |
| F12 | 服务端验证 | 走棋合法性在服务端验证 | 非法走法被拒绝 |
| F13 | 胜负判定（在线） | 服务端判定胜负（吃将/困毙/认输/断线/超时） | 游戏正确结束 |
| F14 | 残局挑战 | 残局列表、提示、重试、进度保存 | 可完成并推进残局 |
| F15 | 好友房间 | 创建/加入房间码 | 双方可通过房间码进入同局 |
| F16 | 观战 | 随机观战与观战列表 | 可进入观战并正常离开 |
| F17 | 复盘 | 历史对局查询与复盘分析 | 可查看历史并逐步回放 |

## 3. 技术约束

| 项目 | 要求 |
|------|------|
| 前端构建 | Vite + TypeScript (strict) |
| 棋盘渲染 | Canvas API |
| AI 实现 | 本地 AI（minimax / alpha-beta） |
| 在线通信 | 原生 WebSocket |
| 后端 | 共享后端 Express + WebSocket（端口 20008） |
| 视角规则 | 玩家永远在下方，敌人在上方 |

## 4. 界面流程

```
模式选择页 → 单机对战页 → 结束弹窗
模式选择页 → 残局挑战页 → 结束弹窗
模式选择页 → 时间档位选择 → 匹配队列 → 在线对战页 → 结束弹窗
模式选择页 → 好友房间页 → 在线对战页 → 结束弹窗
模式选择页 → 历史对局页 → 复盘页
模式选择页 → 观战页 → 在线观战页
```

## 5. 验收标准

### 单机模式
- [ ] 可通过浏览器访问前端页面
- [ ] 可选择红方或黑方开始单机对局
- [ ] 玩家无论选红还是黑，自己的棋子都显示在下方
- [ ] 所有棋子走法规则正确
- [ ] 将军与胜负判定正确
- [ ] 悔棋 / 重开 / 认输可用

### 残局 / 历史 / 观战
- [ ] 残局列表可打开并启动题目
- [ ] 提示、重试、进度保存可用
- [ ] 历史对局可查询并进入复盘
- [ ] 复盘可逐步浏览棋谱
- [ ] 随机观战与观战列表可用

### 在线模式
- [ ] 可选择在线模式进入时间档位选择
- [ ] 匹配到对手后双方进入对局
- [ ] 走棋实时同步
- [ ] 非法走法被服务端拒绝
- [ ] 胜负判定正确
- [ ] 认输 / 和棋可用
- [ ] 对手断线时有提示并可重连

### 通用
- [ ] 前端 TypeScript 编译无错误
- [ ] 通过外部域名访问正常
