# 五子棋需求文档

## 功能需求

### 核心功能
| # | 需求 | 优先级 | 验收标准 |
|---|------|--------|----------|
| F01 | 15×15 棋盘渲染 | P0 | Canvas 绘制网格线、星位标记 |
| F02 | 黑白棋子显示 | P0 | 径向渐变，视觉区分黑白 |
| F03 | 鼠标点击落子 | P0 | 点击交叉点落子，非交叉点忽略 |
| F04 | 胜负判定 | P0 | 五连珠检测（四方向），正确判定胜利 |
| F05 | 平局检测 | P0 | 棋盘满无五连时判定平局 |
| F06 | 人机对战 | P0 | AI 自动响应，有基本攻防 |
| F07 | 双人对战 | P1 | 两人轮流在同一设备落子 |
| F08 | 悔棋功能 | P1 | 撤销最后一步（人机模式撤销两步） |
| F09 | 重新开始 | P1 | 重置棋盘，保留游戏模式 |
| F10 | 最后落子标记 | P1 | 红色圆点标记最后落子位置 |
| F11 | 胜利连线高亮 | P1 | 连珠五子画线高亮 |
| F12 | 游戏状态提示 | P1 | "轮到你" / "AI思考中" / "你赢了" 等 |

### AI 算法
| # | 需求 | 优先级 | 验收标准 |
|---|------|--------|----------|
| A01 | Negamax + Alpha-Beta | P0 | 搜索深度 3，响应 < 2s |
| A02 | 棋型评估函数 | P0 | 识别连五/活四/冲四/活三/眠三/活二等棋型 |
| A03 | 候选点优化 | P0 | 邻域半径 2 格，最多 50 个候选点 |
| A04 | 防守优先 | P1 | 对方威胁权重 ×1.2 |
| A05 | 必胜/必防快判 | P1 | 跳过搜索直接返回 |

### 移动端适配
| # | 需求 | 优先级 | 验收标准 |
|---|------|--------|----------|
| M01 | 触摸落子 | P0 | touchstart 事件支持 |
| M02 | Canvas 自适应 | P0 | 不溢出屏幕，保持正方形 |
| M03 | 按钮触摸热区 | P1 | 最小 44×44px |
| M04 | viewport 适配 | P0 | 禁止缩放，全屏显示 |

## 技术约束
- TypeScript strict 模式
- 无第三方 UI 框架，纯 Canvas + CSS
- 端口规范：开发 20009，生产 20008
- 前端构建产物部署到 `/root/projects/static/gomoku/`
